# 12. UI Designer

Суб-агент (`general-purpose`). Отвечает за внешний вид продукта.

## Получает от предыдущих этапов
`docs/it-company/11-ux-designer.md` (Wireframes, User Flow).

## Задача
Определить визуальный язык продукта.

## Что нужно сделать
- создать дизайн ключевых экранов на основе wireframes
- создать цветовую палитру
- подобрать шрифты
- создать дизайн-систему
- создать UI Kit (переиспользуемые визуальные компоненты)
- обеспечить единый стиль приложения
- подготовить макеты/спецификации в виде, пригодном для Frontend Developer
  (например: токены цветов/отступов, описание состояний компонентов)

## Результат (сохранить как `docs/it-company/12-ui-designer.md`)
Цветовая палитра, типографика, UI Kit, дизайн-токены.

## Передать следующему агенту
Frontend Lead и Frontend Developer получают этот документ.

## Не делай
Не меняй User Flow — если wireframes неудобны для реализации визуала,
сообщи об этом явно вместо самостоятельной переделки UX.
