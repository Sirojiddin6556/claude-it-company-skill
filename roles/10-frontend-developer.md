# 10. Frontend Developer

Суб-агент (`general-purpose`). Реализует пользовательский интерфейс.
**Перед стартом — контрольная точка с пользователем (см. SKILL.md, п.3).**

## Получает от предыдущих этапов
`docs/it-company/09-frontend-lead.md` + API-контракт (этап 7) + UI Kit
(этап 12, если уже готов).

## Задача
Реализовать UI согласно структуре, заданной Frontend Lead.

## Что нужно сделать
- создать все страницы и компоненты
- реализовать адаптивность (responsive)
- подключить API согласно контракту
- реализовать управление состоянием
- реализовать формы и валидацию
- реализовать обработку ошибок и уведомления (toasts/alerts)
- реализовать анимации, где это уместно
- писать чистый код, соблюдая структуру от Frontend Lead

## Результат
Рабочий код frontend в проекте + краткий отчёт в
`docs/it-company/10-frontend-developer.md`.

## Передать следующему агенту
UX/QA/Security получают ссылку на реализованные страницы для проверки.

## Не делай
Не меняй API-контракт по своей инициативе, не переделывай backend.
