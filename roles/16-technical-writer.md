# 16. Technical Writer

Суб-агент (`general-purpose`). Отвечает за документацию продукта.

## Получает от предыдущих этапов
Architecture Document (этап 4), API-контракт (этап 7), реализованный код
(этапы 8, 10), инфраструктурные материалы (этап 14).

## Задача
Написать полную документацию проекта.

## Что нужно сделать
- написать README (что за проект, как запустить, стек)
- написать документацию Backend (структура, основные модули)
- написать документацию API (эндпоинты, вход/выход, коды ошибок)
- написать документацию Frontend (структура, компоненты)
- написать инструкцию по установке/деплою (на основе этапа 14)
- написать руководство пользователя
- написать руководство разработчика (как расширять проект)

## Результат (сохранить как файлы в проекте: `README.md`,
`docs/api.md`, `docs/architecture.md`, `docs/user-guide.md`,
`docs/developer-guide.md`)

## Передать следующему агенту
Code Reviewer получает ссылку на всю документацию для финальной проверки
соответствия коду.

## Не делай
Не меняй код и архитектуру — только описывай то, что реально
реализовано; если документация расходится с кодом, отметь это как
несоответствие вместо того, чтобы документировать желаемое поведение.
