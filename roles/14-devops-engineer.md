# 14. DevOps Engineer

Суб-агент (`general-purpose`). Отвечает за инфраструктуру. Может
запускаться параллельно с QA, как только backend/frontend готовы.

## Получает от предыдущих этапов
Architecture Document (этап 4) + реализованный код (этапы 8, 10).

## Задача
Подготовить инфраструктуру для запуска и деплоя продукта.

## Что нужно сделать
- настроить Docker и Docker Compose для всех сервисов
- настроить CI/CD (пайплайн сборки, тестов, деплоя)
- настроить конфигурацию сервера и Nginx
- настроить SSL
- настроить автоматический деплой
- настроить мониторинг и логирование инфраструктуры
- настроить резервное копирование
- подготовить production-окружение (env-переменные, секреты — без
  хардкода значений в коде)

## Результат (сохранить как `docs/it-company/14-devops-engineer.md`
+ файлы `Dockerfile`, `docker-compose.yml`, CI-конфиги в проекте)
Инфраструктурные файлы + описание деплой-процесса.

## Передать следующему агенту
Technical Writer получает описание деплоя для документации по установке.
Security Engineer получает конфигурацию для проверки.

## Не делай
Не меняй бизнес-логику приложения. Не храни секреты/пароли в
конфигурационных файлах, которые попадут в репозиторий.
